# IT 04-1 Registration Form

A Next.js + Prisma + PostgreSQL registration form satisfying the IT 04-1 spec: capture First/Last name, Email (email format), Phone (phone format), Profile picture (stored as Base64), Birth Day (dd/mm/yyyy), Occupation (combo box, mock data), and Sex (Male/Female); validate every field before saving; **Save** persists to the DB and shows the DB-generated ID with a "save data success" message, then clears the form; **Clear** resets the form without saving.

## Stack

- Next.js 14.2.5 (App Router, TypeScript)
- Prisma 5 (ORM)
- PostgreSQL 16
- Docker / Docker Compose

## Run (one command)

```
docker compose up --build
```

Then open **http://localhost:3000**.

This single command starts Postgres, waits for it to become healthy, builds the web image, applies the Prisma schema to the database (creating the `persons` table), and starts the Next.js server. The first run pulls the `postgres:16-alpine` and `node:20-bookworm-slim` images and installs npm dependencies, so it can take a few minutes; subsequent runs are fast via Docker's build cache.

Stop / reset:

```
docker compose down       # stop containers, keep DB data
docker compose down -v    # stop containers and wipe the DB volume
```

## API

`POST /api/persons`

Request body (JSON):

```json
{
  "firstName": "string",
  "lastName": "string",
  "email": "string (email format)",
  "phone": "string (phone format)",
  "birthDay": "dd/mm/yyyy",
  "occupation": "string",
  "sex": "Male | Female",
  "profileBase64": "string (Base64-encoded image)"
}
```

Responses:

- `201 Created` — `{ "id": <number>, "message": "save data success" }`
- `400 Bad Request` — `{ "errors": { "<field>": "<message>", ... } }`

## Database schema

Table `persons` (Prisma model `Person`, see `prisma/schema.prisma`):

| Column          | Type     | Notes                              |
|-----------------|----------|-------------------------------------|
| `id`            | integer  | primary key, generated by Postgres (autoincrement) |
| `firstName`     | text     |                                      |
| `lastName`      | text     |                                      |
| `email`         | text     |                                      |
| `phone`         | text     |                                      |
| `birthDay`      | date     |                                      |
| `occupation`    | text     |                                      |
| `sex`           | text     |                                      |
| `profileBase64` | text     | Base64-encoded profile image        |
| `createdAt`     | timestamp | defaults to now()                  |

`id` is never sent by the client — it's generated by the database and returned in the Save response.

## Local development without Docker

Requires a local PostgreSQL instance matching the connection string in `.env`.

```
npm install
npx prisma db push
npm run dev
```
